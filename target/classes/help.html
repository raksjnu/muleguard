MuleGuard - Help & Documentation

MuleGuard is a comprehensive static analysis solution designed specifically for MuleSoft applications, ensuring adherence to organizational coding standards, optimal security practices, and readiness for future migrations.

üìã What MuleGuard Validates

MuleGuard provides thorough analysis across all core project components:

  • Maven configurations: pom.xml validation
  • Mule runtime manifests: mule-artifact.json structure
  • Mule flows & sub-flows: XML configuration files
  • DataWeave scripts: .dwl files
  • Environment-specific property files: .properties and .policy files

üìö Rule Types

Code Rules (12 types)

  • Token Search: Required/Forbidden tokens in code files
  • XML XPath: XPath expression validation (exists/not exists)
  • XML Attributes: Attribute validation (exists/not exists)
  • XML Element Content: Element content validation (required/forbidden)
  • POM Validation: Maven dependency validation (required/forbidden)
  • JSON Validation: JSON structure validation (required/forbidden)

Config Rules (6 types)

  • Mandatory Substring Check: Environment-specific token validation
  • Property Value Checks: Mandatory and optional property validation
  • Generic Token Search: Advanced token search with environment filtering
  • Client ID Validator: Secure property and client ID mapping validation
  • Generic Property File Check: Flexible property file validation

üöÄ How to Use

Command Line

java -jar muleguard-1.0.0-jar-with-raks.jar -p /path/to/your/mule-projects

GUI Mode

Run start-gui.bat (Windows) or start-gui.sh (Mac/Linux)

Opens browser at http://localhost:8080

üìñ Documentation

Detailed rule type documentation is available in the docs/ folder:

  • Each rule type has its own documentation file

‚öôÔ∏è Configuration

All rules are configured in src/main/resources/rules/rules.yaml

  • Enable/Disable rules: Set enabled: true/false
  • Severity levels: CRITICAL, HIGH, MEDIUM, LOW
  • Custom parameters: Each rule type supports specific parameters
  • Environment filtering: Target specific environments (DEV, QA, PROD)

üìä Report Output

  • CONSOLIDATED-REPORT.html: Dashboard for all scanned APIs
  • Individual reports: Detailed results for each API
  • checklist.html: Complete list of all validation rules
  • Excel reports: .xlsx format for data analysis

üí° Tips

  • Click on individual API reports for detailed rule-by-rule results

  • Review the docs/ folder for rule-specific configuration examples
